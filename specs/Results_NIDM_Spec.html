NI-R Core Structures

The concepts found in the core of NI-Results are introduced in the rest of this section. They are summarized in Table 2, where they are categorized as type or relation. The first column lists concepts, the second column indicates whether a concept maps to a type or a relation, whereas the third column contains the corresponding name, as it appears in Figure 1. Names of relations have a verbal form in the past tense to express what happened in the past, as opposed to what may or will happen. In the core of PROV, all relations are binary.

A ModelParametersEstimation ◊is an Activity defined as The process of estimating the parameters of a general linear model from the available data..

ClusterStatistic

A ClusterStatistic ◊is an Entity defined as Statistic defined at the cluster-level in an excusion set. In SPM it is defined as the cluster size. .

A ClusterStatistic ◊ has:
  • prov:label: an OPTIONAL label describing the ClusterStatistic.
  • clusterSizeInVoxels: an OPTIONAL clusterSizeInVoxels describing the Number of voxels that make up the cluster..
  • clusterLabelId: an OPTIONAL clusterLabelId describing the Integer associated with a particular cluster as specified in the clusterLabelsMap..
  • clusterSizeInResels: an OPTIONAL clusterSizeInResels describing the Number of resels that make up a cluster..
  • pValueUncorrected: an OPTIONAL pValueUncorrected describing the A p-value reported without correction for multiple testing. .
  • pValueFWER: an OPTIONAL pValueFWER describing the "A quantitative confidence value resulting from a multiple testing error correction method which adjusts the p-value used as input to control for Type I error in the context of multiple pairwise tests".
  • qValueFDR: an OPTIONAL qValueFDR describing the P-value adjusted for the multiple testing, controlling for the False Discovery Rate.
