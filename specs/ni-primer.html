This document provides an introduction to the NIDASH Data Model (NI-DM) for neuroimaging data sharing and exchange. NI-DM extends the [[PROV]] for capturing provenance information on the Web, to provide a collection of interoperable domain models for Protocol, Experiment, Workflow, and Derived Data Components. Each Component provides one or more models that captures relevant information within the given scope.

The document separates the overall NI-DM graph into smaller sub-graphs for clarity in understanding how to construct the various entities and relationships.

Introduction

The NI-DM is an ongoing effort to develop an extensible model for neuroimaging derived data with support for rich, queryable provenance and metadata. Our approach with NI-DM is to capture, in a single data model, the different components of a research activity (e.g., participants, researchers, software, and hardware) and their relations. With the observation that activities represent transformations of data, it is easy to recognize the close link between data and provenance and that these information should be captured together. Fig. 1 provides a high level view of how NI-DM can be used to explicitly model research activities in the context of provenance, which provides a computable representation that supports rich query capabilities.

The single conformance class described in this specification is the NI-DM Core Specifictaion.
